# Roadmap

Updated: 2026-06-06

## Phase 0: Foundation

Goal: define the course spine and product hypothesis.

Tasks:

- Fix notation for cube pieces, moves, face names, permutations, and orientations.
- Decide whether videos use Singmaster notation, Chinese move names, or both.
- Create a 2x2-first explanation path.
- Create a 3x3 expansion path.
- Design the numbered cube prototype.
- Build a source list for mathematical claims.

Exit criteria:

- One-page positioning statement.
- First 10 video titles.
- First prototype cube design.
- Draft outline of the PDF notes.

## Phase 1: Audience Test

Goal: find which content format has pull.

Tasks:

- Publish short videos on high-curiosity topics.
- Publish at least one rigorous longer proof video.
- Collect comments, questions, and confusion points.
- Track saves, follows, completion rate, and product interest.

Suggested first topics:

- "为什么三阶魔方不能只交换两个角块？"
- "魔方到底有多少种状态？"
- "群论不是玄学：魔方每一步都是一个置换"
- "二阶魔方为什么是学习群论的好入口？"
- "God's number 到底是什么意思？"

Exit criteria:

- At least 5 public videos.
- Clear evidence of which audience segment responds.
- A list of repeated viewer questions.

## Phase 2: Learning Kit MVP

Goal: sell a small batch without overcommitting inventory.

Tasks:

- Create the first PDF note pack.
- Create a simple product page or pinned post.
- Build 10-50 prototype bundles.
- Test price, packaging, and fulfillment workflow.
- Document customer feedback.

Exit criteria:

- First paid orders.
- Real landed cost per bundle.
- Refund/quality issues understood.
- Decision on whether RMB 38 is sustainable.

## Phase 3: Course System

Goal: turn scattered videos into a coherent course.

Tasks:

- Write full lecture notes for 2x2.
- Write full lecture notes for 3x3 state counting and invariants.
- Make diagrams, animations, and proof slides.
- Package the content into a playlist and PDF.
- Create a stable errata file for mathematical corrections.

Exit criteria:

- Complete 2x2 module.
- Complete 3x3 state-counting module.
- First version of PDF course notes.
