## 0. The Result

The standard 3x3 Rubik's cube has

```text
43,252,003,274,489,856,000
```

legal states.

Equivalently,

```text
8! * 3^7 * 12! * 2^11 / 2
= 43,252,003,274,489,856,000.
```

This number is not obtained by counting stickers independently. It comes from counting cubies, orientations, and three hidden constraints.

## 1. Public Intuition First

The 3x3 cube has:

- 8 corner cubies.
- 12 edge cubies.
- 6 center cubies.

In the standard model, the centers determine the color frame, so we treat them as fixed reference pieces.

Naively:

- The 8 corners can be placed in `8!` ways.
- Each corner can be twisted in `3` ways, giving `3^8` corner orientations.
- The 12 edges can be placed in `12!` ways.
- Each edge can be flipped in `2` ways, giving `2^12` edge orientations.

So the naive mechanical count is:

```text
8! * 3^8 * 12! * 2^12.
```

But most of these arrangements cannot be reached by legal face turns.

There are three hidden laws:

1. The total corner twist must be `0 mod 3`.
2. The total edge flip must be `0 mod 2`.
3. The corner permutation and edge permutation must have the same parity.

These remove a factor of:

```text
3 * 2 * 2 = 12.
```

Therefore:

```text
(8! * 3^8 * 12! * 2^12) / 12
= 8! * 3^7 * 12! * 2^11 / 2.
```

## 2. Mathematical Setup

### Definition 2.1: Bijection

A bijection from a finite set `A` to itself is a rearrangement of the elements of `A`.

For the Rubik's cube, a move rearranges cubies without duplicating or deleting them. This is why permutations naturally appear.

### Definition 2.2: Symmetric Group

Let `S_n` be the group of all permutations of `{1, 2, ..., n}`.

We will use:

- `S_8` for the corner cubies.
- `S_12` for the edge cubies.

### Definition 2.3: Cycle Notation

The cycle

```text
(a1 a2 ... ak)
```

means:

```text
a1 -> a2,
a2 -> a3,
...
ak -> a1.
```

Example:

```text
(1 3 5)
```

sends `1` to `3`, `3` to `5`, and `5` back to `1`.

### Definition 2.4: Sign Of A Permutation

Every permutation has a parity:

- even: sign `+1`;
- odd: sign `-1`.

A transposition, meaning a swap of exactly two objects, is odd.

A `k`-cycle has sign:

```text
(-1)^(k-1).
```

So a 4-cycle is odd.

## 3. The Cube Move Group

Let `G` be the group of legal 3x3 moves.

The elements of `G` are move sequences generated by face turns:

```text
U, D, L, R, F, B
```

and their inverses.

Two move sequences are considered the same element of `G` if they have the same effect on every cubie position and orientation.

The identity element is the empty move.

The inverse of a move sequence is the same sequence performed backward with every turn reversed.

## 4. Encoding A Cube State

A 3x3 cube configuration can be encoded as:

```text
(sigma, tau, x, y)
```

where:

- `sigma in S_8` records the permutation of the 8 corner cubies.
- `tau in S_12` records the permutation of the 12 edge cubies.
- `x = (x_1, ..., x_8) in (Z/3Z)^8` records the orientations of the 8 corners.
- `y = (y_1, ..., y_12) in (Z/2Z)^12` records the orientations of the 12 edges.

Interpretation:

- `x_i = 0, 1, 2` tells how the `i`-th corner is twisted relative to the solved orientation.
- `y_j = 0, 1` tells whether the `j`-th edge is unflipped or flipped relative to the solved orientation.

The solved cube is:

```text
(1, 1, 0, 0)
```

where `1` means the identity permutation and `0` means the all-zero orientation vector.

## 5. The Legal Configuration Theorem

### Theorem

A configuration

```text
(sigma, tau, x, y)
```

is legal, meaning reachable from the solved cube by legal face turns, if and only if:

```text
sgn(sigma) = sgn(tau),
sum_i x_i = 0 mod 3,
sum_j y_j = 0 mod 2.
```

This is the central theorem of the state-count lesson.

## 6. Proof: Why The Three Conditions Are Necessary

We prove that every legal move preserves the three conditions.

### 6.1 Corner And Edge Parity Must Match

Consider a single face turn, for example `R`.

It cycles 4 corner cubies and 4 edge cubies.

At the permutation level, a face turn contributes:

```text
one 4-cycle on corners,
one 4-cycle on edges.
```

A 4-cycle has sign:

```text
(-1)^(4-1) = -1.
```

So a single face turn changes both corner parity and edge parity.

Therefore the two signs remain equal:

```text
sgn(sigma) = sgn(tau).
```

Since every legal move is a product of face turns, this equality is preserved by every legal move sequence.

### 6.2 Total Corner Twist Is Conserved Modulo 3

Each corner has three possible orientations.

We encode these as:

```text
0, 1, 2 mod 3.
```

For any basic face turn, the corner twists change in a way whose total sum is unchanged modulo 3.

Thus:

```text
sum_i x_i mod 3
```

is an invariant of legal moves.

The solved cube has total corner twist `0`, so every legal configuration satisfies:

```text
sum_i x_i = 0 mod 3.
```

### 6.3 Total Edge Flip Is Conserved Modulo 2

Each edge has two possible orientations:

```text
0 or 1 mod 2.
```

For any basic face turn, the number of edge flips changes by an even amount.

Thus:

```text
sum_j y_j mod 2
```

is an invariant of legal moves.

The solved cube has total edge flip `0`, so every legal configuration satisfies:

```text
sum_j y_j = 0 mod 2.
```

This proves necessity.

## 7. Proof: Why The Three Conditions Are Sufficient

Now suppose a configuration satisfies:

```text
sgn(sigma) = sgn(tau),
sum_i x_i = 0 mod 3,
sum_j y_j = 0 mod 2.
```

We must show it is reachable from the solved cube.

Equivalently, if the cube is currently in that configuration, we must show there is a legal move sequence that solves it.

The proof is constructive.

### Step 1: Put Corners Into Correct Positions

We use the fact that the corner-position map

```text
phi_corner: G -> S_8
```

is onto.

Meaning:

For every corner permutation in `S_8`, there is some legal move sequence whose effect on corner positions is exactly that permutation.

So we can choose a move sequence that sends the corner permutation `sigma` back to the identity.

After this step, the configuration has the form:

```text
(1, tau', x', y').
```

### Step 2: Fix Corner Orientations

The condition

```text
sum_i x_i = 0 mod 3
```

means the corner twists balance globally.

There are legal move sequences that twist two corners in opposite directions while preserving all corner positions.

Using such moves repeatedly, we can correct one wrongly twisted corner at a time.

Because the total twist is `0 mod 3`, the final corner is automatically corrected when the previous seven have been corrected.

After this step, the configuration has the form:

```text
(1, tau'', 0, y'').
```

### Step 3: Put Edges Into Correct Positions

At this point, the corners are solved.

Since the original configuration satisfied:

```text
sgn(sigma) = sgn(tau),
```

and now `sigma = 1`, the remaining edge permutation must be even.

Every even permutation can be written as a product of 3-cycles.

There are legal move sequences that cycle three edge cubies while preserving solved corners.

Therefore we can correct the remaining edge permutation without disturbing the corners.

After this step, the configuration has the form:

```text
(1, 1, 0, y''').
```

### Step 4: Fix Edge Orientations

The condition

```text
sum_j y_j = 0 mod 2
```

means the number of flipped edges is even.

There are legal move sequences that flip two edge cubies while preserving all cubie positions and solved corner orientations.

Using these moves repeatedly, we can correct flipped edges two at a time.

After this step, the configuration is:

```text
(1, 1, 0, 0).
```

This is the solved cube.

Therefore every configuration satisfying the three conditions is legal.

## 8. Counting Legal States

Now count all configurations satisfying the theorem.

### 8.1 Corner Positions

The 8 corners can be permuted freely:

```text
8!
```

### 8.2 Corner Orientations

There are 8 corners, each with 3 possible orientations:

```text
3^8.
```

But the sum must be:

```text
0 mod 3.
```

Once 7 corner orientations are chosen, the 8th is forced.

So:

```text
3^7.
```

### 8.3 Edge Positions

The 12 edges can be permuted:

```text
12!
```

### 8.4 Edge Orientations

There are 12 edges, each with 2 possible orientations:

```text
2^12.
```

But the sum must be:

```text
0 mod 2.
```

Once 11 edge orientations are chosen, the 12th is forced.

So:

```text
2^11.
```

### 8.5 Parity Match

Among all choices of corner and edge permutations, exactly half satisfy:

```text
sgn(sigma) = sgn(tau).
```

So we divide by 2.

### 8.6 Final Formula

Therefore:

```text
8! * 3^7 * 12! * 2^11 / 2
= 43,252,003,274,489,856,000.
```

## 9. What This Means

The cube is not hard because the stickers are many.

It is hard because the legal states form a highly structured subset of all imaginable arrangements.

The group-theoretic lesson is:

> A puzzle is not just a collection of positions. A puzzle is a set of positions together with the moves that connect them.

The hidden laws are not psychological tricks. They are invariants.
